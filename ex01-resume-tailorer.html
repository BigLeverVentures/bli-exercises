Big Lever Institute · Exercise 01

Resume Tailorer — build a reusable Claude Project for any JD

In the next 25–45 minutes you'll go from "paste a resume into Claude" to a permanent Claude Project that tailors a resume and cover letter to any job description in about 30 seconds per run. What you'll actually learn is the 3-layer architecture under it — usable for any repetitive task you do.

Step 0

Before you start

Have two things ready:

  1. Your current resume — text or a document you can paste from.
  2. A target job description — ideally a role you'd actually apply for. The more real, the better the output.

You'll also need a Claude account (free tier works). Other LLMs work, but we default to Claude for consistency. Estimated time: 25–45 minutes.

Step 1 · Show

Produce your first tailored output

Open claude.ai in a new tab and start a new chat. Copy the prompt below, paste it into Claude, replace the two bracketed sections with your resume and the target JD, then send.

Starter prompt — copy, paste, fill in brackets, send
I'm building a personalized resume-tailoring system, starting with a first real tailored output for a specific job.

**My current resume:**
[PASTE YOUR RESUME HERE]

**The target job description:**
[PASTE THE JD HERE]

Using my resume above and the JD above, produce two outputs — a tailored resume (1–2 pages) and a matching cover letter (≤350 words). Apply all of the following rules.

**General principles**

1. **Match the employer's tone.** Use the same linguistic register, keywords, and recurring phrases found in the JD. Mirror whether the employer writes in corporate, academic, or startup language. Use the company's mission statements or recurring verbs to frame the narrative — as if the resume and letter were written from within the organization.
2. **Prioritize alignment.** Only include achievements relevant to the JD's top 3 themes. Re-order experience by relevance, not chronology — most relevant role first. Use the employer's exact keyword phrasing wherever credible (for ATS matching).
3. **Proof-driven bullets.** Every bullet = **Action + Context + Quantified Impact**. Example: "Reduced supplier lead times by 30% through AI-driven forecasting, accelerating product launch by two months." At least 70% of bullets must include a specific number, percentage, timeline, or scope marker. If you need to estimate a metric, flag it with [est.] so I can verify later.
4. **Modern resume standards.** Bullets concise (maximum 2 lines each). **No pronouns**; lead every bullet with a strong verb. Preferred verbs: built, led, scaled, negotiated, architected, reduced, increased, launched, shipped, delivered. White space, not dense blocks. **No filler phrases** — no "responsible for," "helped with," "duties included."
5. **Voice.** Confident but not self-promotional. Clarity and credibility over buzzwords. Replace buzzwords with demonstrable proof of results.

**Resume structure (1–2 pages total)**

1. **Header** — Name, target title, location, contact, LinkedIn.
2. **Executive Summary** — 3–4 lines tailored to this specific role.
3. **Key Achievements** — 3 concise bullets most relevant to the JD's top themes.
4. **Professional Experience** — ordered by relevance to this role, not chronology.
5. **Education & Affiliations** — condensed.
6. **Selected Mentorship / Leadership Roles** — if applicable.

**Cover letter structure (≤350 words, 3–4 paragraphs)**

- **Opening** — establish immediate relevance by mirroring the role title and the company's mission or value language from the posting.
- **Middle (1–2 paragraphs)** — 2–3 proof points that directly map to the JD's top themes; each quantified.
- **Closing** — alignment with company goals and cultural tone; brief forward-looking contribution statement.

**Process (follow these 6 steps before producing output)**

1. Identify the top 3–5 keywords, required skills, or recurring themes in the JD.
2. Determine the target persona — corporate executive, startup builder, academic innovator, or hybrid.
3. Select relevant proof points from my resume.
4. Rewrite phrasing to match the employer's diction and rhythm.
5. Produce the tailored resume and matching cover letter.
6. **Verify before delivering:** every bullet follows Action + Context + Quantified Impact; ≥70% of bullets include a quantified metric; no pronouns; no filler phrases; bullets ≤2 lines; resume ≤2 pages; cover letter ≤350 words; tone matches the JD; JD keywords appear verbatim where credible.

Present the tailored resume first, then the cover letter. Label each clearly.

Wait for the output. Read the resume and cover letter end-to-end. The first pass should be close to something you could actually send — if it's not, Step 2 will sharpen it.

Step 2 · Show (refine)

Sharpen the first pass

Most first passes need one round of sharpening. Pick whichever of these two most closely matches what's off, copy it, and run it in the same Claude chat.

Option A — Sharpen themes & keyword alignment
Looking at the JD again, what are the top 3 themes the employer emphasizes? For each theme, tell me which of my bullets best proves it, and flag which JD keywords appear verbatim in my resume and which don't. Then rewrite the resume to foreground those bullets first and trim lower-relevance items. Adjust the cover letter's middle paragraphs to mirror the same three themes.
Option B — Adjust the voice register
The current resume and letter read [too formal / too casual / too generic — pick the one that applies]. Rewrite both to sound as if written from inside a [startup / corporate / academic / hybrid] organization — using their diction, verbs, and framing — without adding any claims I can't substantiate from the material I gave you.

Iterate once — twice if needed. When you have a version you're satisfied with, continue.

Step 3 · Give (encapsulate)

Convert what worked into a reusable Protocol

In the same Claude chat, run the prompt below. Claude will write the Custom Instructions you'll install in the next step.

Encapsulation prompt — copy, paste, send
Now let's convert what we just did into a reusable system.

Draft a complete set of **Custom Instructions** I can paste into a new Claude Project. The Project will be named "Resume Writer." The Custom Instructions should encode:

1. **Purpose** — for every new JD I provide, generate a tailored resume (1–2 pages) + cover letter (≤350 words) as if written from inside the organization.
2. **Source files** — reference two files uploaded to the Project: my master resume, and an optional narrative/voice document.
3. **Five principles** —
   (a) match the employer's tone, keywords, and phrasing;
   (b) proof-driven bullets in Action + Context + Quantified Impact format;
   (c) prioritize alignment to the JD's top 3 themes; use employer keywords exactly where credible (for ATS);
   (d) modern resume standards — concise bullets, strong verbs, 70%+ quantified, re-order by relevance not chronology, white space;
   (e) confident tone, not self-promotional; clarity and credibility over buzzwords.
4. **Resume structure** — Header; 3–4 line Executive Summary; Key Achievements (3 bullets relevant to job); Professional Experience (relevance-ordered); condensed Education & Affiliations; Selected Mentorship/Leadership (if applicable).
5. **Cover letter structure** — Opening mirroring role title + company mission; Middle with 2–3 quantified proof points; Closing with alignment + forward-looking contribution statement.
6. **Per-instance process** — six steps: (i) identify top 3–5 JD keywords/themes; (ii) determine target persona; (iii) select proof points from source docs; (iv) rewrite phrasing to match employer diction; (v) produce resume + cover letter; (vi) verify Action→Result and quantified bullets.
7. **Style checklist** — resume 1–2 pages; cover letter ≤350 words; proof-driven verbs (built, led, scaled, negotiated, architected, reduced, increased); no filler ("responsible for," "helped with").

Format the output as **one continuous block of Custom Instructions text, ready to paste directly** into a Claude Project's Custom Instructions field. Do not include commentary before or after the block.

Claude will produce a block of text. Keep it visible — you'll paste it in the next step.

Step 4 · Give (install)

Install your Protocol as a Claude Project

You now have a reusable Protocol. Install it so you never have to repeat Steps 1–3 again.

Deliverable — your permanent Resume Writer
  1. Go to claude.ai → Projects → New Project.
  2. Name it Resume Writer.
  3. Under Custom Instructions, paste the block Claude just produced. Save.
  4. Under Project Knowledge, upload:
    • Your master resume (text file or document)
    • Any narrative/voice document you want Claude to draw from (optional)
  5. Start a new chat inside the Project.
  6. Paste the invoking prompt below, then paste any JD beneath it.
Invoking prompt — your new one-sentence command
Using my Resume Writer Protocol, generate a tailored resume and cover letter for the job description below:

[PASTE JD]

Test it on a real JD now. Compare this output to the one you got in Step 1. What changed — and what's the same? That's the signal that the pattern works.

Step 5 · Tell

How this actually works

What you just built follows a simple architecture. It works for any task where you repeat similar work with varying inputs.

Layer 1 — Stable System: your Custom Instructions. The rules of the game. Tone, format, structure, style. These rarely change. When they do, you edit the Protocol directly in the Project's Custom Instructions.
Layer 2 — Stable Context: your source docs. The facts Claude can cite. Your career, your voice, your achievements. These change only when your life changes.
Layer 3 — Volatile Input: each new JD. The only thing that varies per run. You paste it; the system does the rest.

This is why one paste produces a tailored output in about 30 seconds: Claude isn't guessing your background, your voice, or the rules. All three are already loaded. You're pasting one variable into a configured machine.

Two implications:

  • Update your Protocol when your career shifts, not when small details change. Edit it directly in the Project's Custom Instructions. Log the reason in a comment at the top — your future self will want to know.
  • Any recurring task that varies case-by-case is a candidate for this same 3-layer pattern.
Step 6 · ExtendOptional · ~15 min

Apply this to one more task

Let's prove the pattern by applying it somewhere else. Here's a worked example, then a prompt you can run to build your own second Protocol.

Worked example — Outreach Tailor

Customize cold and networking outreach per recipient.

  • Stable system (rules): keep under 5 sentences; open with a specific connection, not a pleasantry; include one piece of evidence you did your homework on the recipient; end with exactly one specific ask; no hype verbs ("amazing," "excited," "passionate").
  • Stable context (about you): short bio, the roles/opportunities you're targeting, your top three proof points.
  • Volatile input (per message): the recipient's profile + what prompted your reach-out.
  • Output: a 5-sentence outreach message tuned to them.

Same three layers, different domain. Once installed, sending 20 well-tailored outreach messages takes about as long as sending 3 generic ones — and pulls far better reply rates.

Now pick a task of your own — investor updates, weekly team updates, thank-you notes, proposals — and run this prompt in a fresh Claude chat.

Generator prompt — build your second Protocol
I'd like to build a reusable Protocol like my Resume Writer, but for a different task. Here's the task: [describe one repetitive task you do that varies case-by-case — for example, networking outreach, weekly team updates, thank-you notes after interviews, proposals, investor updates].

Using the same 3-layer architecture (stable rules, stable context, volatile input), propose a full set of Custom Instructions I could paste into a new Claude Project.

**Ask me any clarifying questions first** — about the audience, output format, tone, length, must-include elements, things to avoid. Do not guess; ask.

After I answer, produce the Custom Instructions as one continuous block ready to paste directly into a Claude Project. Also suggest what I should upload as source docs and propose a name for the Project.

This is what prompt engineering actually is — writing stable rules that turn a variable input into a consistent, high-quality output. Every Custom Instructions document you build is added leverage.
