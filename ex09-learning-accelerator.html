Big Lever Institute · Exercise 09

Learning Accelerator — build a persistent AI tutor on any topic

In the next two hours you'll turn raw source material into a persistent, adaptive AI tutor on a topic you actually want to master. You'll build it once and return to it every day for four weeks. Then you'll learn the pattern underneath — so you can spin one up for any future topic in about 20 minutes.

Step 0

Before you start

Have three things ready before the 2-hour timer starts:

  1. A topic you want to master. Be specific. Not "supply chain." Try "Supply Chain Risk," "Kubernetes Multi-Region Architecture," or "Massachusetts Nonprofit Fundraising Law."
  2. 3–8 source documents. PDFs, article links, blog posts, YouTube transcripts, whitepapers, course notes. Quality over quantity. If the pool is thin, pad with depth, not breadth.
  3. Your North Star sentence. Finish this stem: "In 4 weeks, I want to be able to _____." Write it down. You'll paste it directly into the prompt in Step 4; your tutor will be anchored to it from the start.

You'll also need a free Google account (for NotebookLM) and a Claude account (free tier works, though Claude memory — enabled in Step 3 — is typically a paid-tier feature). Estimated time: about 2 hours, with an optional 30 minutes for Step 7.

Step 1 · Show

Build your artifact library in NotebookLM

Open notebooklm.google.com in a new tab. Click Create new notebook and name it after your topic (e.g., "Supply Chain Risk"). Click Add source and upload your PDFs, paste article text, or drop YouTube links — one source at a time. Wait 30–60 seconds for processing to finish.

Now generate four artifacts. These are deterministic — you run them once and never return to NotebookLM. If NotebookLM has built-in buttons for Study Guide, Q&A, or Briefing, click those directly. Otherwise, paste the prompts below into NotebookLM's chat pane one at a time.

Artifact 1 · Study Guide
Create a comprehensive study guide for this topic with 3–4 major sections. For each section, include:
- A 2–3 sentence summary of the core idea
- 3–5 key concepts or takeaways
- 2–3 quiz-style questions with answers

Format as markdown. Aim for 2,000–3,000 words total.
Artifact 2 · FAQ
Create a 50–100 item FAQ answering the most important questions someone learning this topic would ask. Order by:
1. Foundational (what is X, why does it matter)
2. Applied (how do I use this, best practices)
3. Pitfalls (common mistakes, how to avoid them)

Format: **Q:** [question] **A:** [1–3 sentence answer]. One per line. No markdown headers.
Artifact 3 · Briefing
Create a 500-word executive briefing for someone who has 10 minutes to understand this topic at a strategic level. Include:
- What this topic is and why it matters
- Top 3–5 strategic implications or applications
- Key metrics or success factors
- 2–3 critical questions to evaluate this in practice

Write in clear, non-technical language. Audience: senior manager or board member.
Artifact 4 · Timeline / Roadmap (skip if your topic has no sequence)
Create a 5–7 phase learning or implementation roadmap. For each phase:
- Name the phase
- 3–4 key milestones or topics to master
- Estimated time to completion
- Prerequisite phases

Format as a markdown outline.

When all four artifacts are generated, export each one to your computer. Select all (Ctrl/Cmd+A), copy (Ctrl/Cmd+C), and paste into plain-text files named: study-guide.txt, faq.txt, briefing.txt, and timeline.txt. Keep them somewhere you can find them — you'll upload them to Claude in Step 5.

Bonus for commute listening: NotebookLM also has an Audio Overview button. Click it if you want a 10–15 min podcast-style walkthrough you can listen to while driving or walking. It's not part of the tutor — just an optional consumption channel.
Step 2 · Tell

How this actually works

You just built the input side of a two-tool learning system. Here's the architecture you're about to complete.

Tool A — NotebookLM (the artifact generator). Takes raw source material, produces structured study artifacts. Deterministic. Runs once. The four files you just exported are now stable context.
Tool B — Claude Projects (the active tutor). Lives in a Project. Shaped by Custom Instructions you'll install in Step 5. Reads the four artifacts as ground truth. Quizzes, scaffolds, and adapts to your pace across sessions.
The pipeline — Tool A's output becomes Tool B's stable context. This is the skill being transferred. You're not using NotebookLM in isolation or Claude in isolation. You're chaining them and shaping Tool B's behavior with a persistent system prompt.

This is the prompt engineering skeleton underneath: Stable System (Custom Instructions) × Stable Context (Project files) × Volatile Input (what you type today). You just built the first two. The third is every tutoring conversation you'll have for the next four weeks.

Why this beats ChatGPT-ing your way through a topic:

  • No context-switching. Everything pinned in one place.
  • Stateful. The tutor remembers where you got stuck, what you know cold, what's still fuzzy.
  • Retrieval practice is enforced. No lecture-mode. Every session is diagnostic → quiz → scaffold → apply.
  • The pattern generalizes. You'll use it again in Step 7.
Step 3 · Give

Turn on Claude memory

This is the difference between a tutor that grows with you and one that starts over every time.

Without memory enabled, your tutor resets at the start of every new session. Every conversation is a blank slate — no recollection of what you struggled with last Tuesday, what you've already mastered, or where you left off. With memory on, the tutor actually gets smarter every time you come back.

Turn it on:

  1. Go to claude.ai.
  2. Click your profile icon (bottom-left), then Settings → Profile (or Personalization).
  3. Find Memory (sometimes labeled Remember context from past chats) and turn it on.
If memory isn't available on your plan: You can still run this exercise. The Custom Instructions you generate in Step 4 will tell Claude to maintain a learner_knowledge_map.md file as a fallback. It still works — it's just not as seamless.
Step 4 · Give (generate)

Create your Project and generate your Custom Instructions

At claude.ai, click Projects in the sidebar, then New Project. Name it Study Partner — [Your Topic] (e.g., "Study Partner — Supply Chain Risk").

Click New Chat inside the Project. Paste the prompt below. Claude will ask you five questions; answer them in the same chat. Claude will then produce two things: your 3-milestone learning plan and a complete block of Custom Instructions.

Encapsulation prompt — paste into a new chat inside your Project
Context: I'm setting up this Claude Project as a persistent tutor to teach me a topic I want to master. Before I install my Custom Instructions, I need them tailored to my learning context, along with a 3-milestone plan.

Ask me the five questions below. Once I've answered all five, produce:
(a) A 3-milestone learning plan for the next 4 weeks
(b) A complete block of Custom Instructions I can paste directly into this Project's Custom Instructions field

Do not generate either until I've answered all five questions.

**Ask me these questions:**

1. **Topic & domain.** What are you teaching me? What field is it in?
   (e.g., "Supply Chain Risk, in operations management")

2. **Prior knowledge.** Rate yourself 1–5 (1 = never heard of it; 5 = expert). What adjacent knowledge do you already have that might help?

3. **North Star — Learning outcome.** What do you want to be able to teach, decide, or do after 4 weeks? Finish this sentence: "In 4 weeks, I want to be able to ___." Be specific. This is my north star — keep it front and center in every session.

4. **Study pace.** How much time per week can you commit? (e.g., "5 hours," "20 min daily," "2 hr Saturdays")

5. **Learning style.** Pick one or two:
   - Quiz-first — test me immediately so I know what I don't know
   - Example-driven — show me how concepts apply in real situations
   - Conceptual — explain the "why" and architecture before application
   - Scenario-based — walk me through realistic problems to solve

**When you generate my Custom Instructions, they must:**
- Anchor every session to my North Star (question 3). Surface it explicitly at the start of each session.
- Enforce Socratic, retrieval-first posture — quiz before explaining.
- Adapt pacing to my knowledge level and available time.
- Reference my Project files (Study Guide, FAQ, Briefing, Timeline) as ground truth.
- Maintain a running "learner knowledge map" — if Claude memory is enabled, use that; if not, maintain it as a file called learner_knowledge_map.md in Project files, updated at the end of each session.
- Structure every session as: diagnostic → quiz → scaffold → deepen → close.
- Prohibit multi-paragraph lecture-mode explanations.

Read the 3-milestone plan Claude produces. If it matches your North Star, move on. If not, ask Claude for one adjustment in the same chat. Keep the Custom Instructions block visible — you're about to paste it.

Step 5 · Give (install)

Install the system and invoke your tutor

You have the ingredients. Install them.

Deliverable — your permanent Study Partner
  1. Paste your Custom Instructions. Open Custom Instructions in your Project (gear icon or Settings link at the top of the Project). Paste the block Claude just produced. Save.
  2. Upload your four artifacts. Find the Project's file library (folder icon; often labeled Project knowledge or Project documents — it's where the Project stores reference files the tutor can read). Click Add files and upload:
    • study-guide.txt
    • faq.txt
    • briefing.txt
    • timeline.txt (if you created one)
    Wait for all uploads to complete.
  3. Invoke your tutor. Click New Chat inside the Project and paste the invoking prompt below.
Invoking prompt — your opener, for this session and every future one
Let's start. I'm ready to begin learning. Run your diagnostic first — find out where I'm starting from. Then we'll pick the first milestone.

Claude will run a diagnostic, ask 2–3 clarifying questions, reference your North Star, surface your 3-milestone plan, and open the first tutoring conversation. From here, your Custom Instructions shape every response — Socratic, adaptive, retrieval-focused.

Step 6 · Give (use)

Work with your tutor — now and for the next four weeks

Use the rest of your 2-hour window to run your first 1–2 tutoring conversations. Let the tutor quiz you, scaffold where you struggle, and close with a recall check. Then come back every day. Here's how to get the most out of it over four weeks:

  • Stay in the chat for the duration of a study session. Let the tutor's diagnostic → quiz → scaffold → deepen → close loop run its course.
  • Deepen on any topic. If a quiz surfaces a gap, ask the tutor to go deeper on that specific concept. It will scaffold, not lecture.
  • Branch into a new chat when you want to explore a subtopic without cluttering your main thread. Your Custom Instructions and files come with you — every chat in this Project is shaped the same way.
  • Resume anytime. Come back tomorrow, next week, or next month. Open the Project, start a new chat (or continue the existing one), and type "Let's pick up where we left off." With memory on, the tutor recalls where you were. Without memory, it reads the knowledge-map file it maintains.
  • Iterate on the tutor itself. If pacing is off or the quizzing is too easy or too hard, tell it — or edit the Custom Instructions directly in Project settings. You designed this system; you can redesign it.

This Project is yours now. Return to it every day for the next four weeks. That's how you master the topic.

Step 7 · ExtendOptional · ~30 min

Build your Learning System Generator

The core deliverable is done — you have a working Study Partner. You can come back to this step anytime. It turns today's one-off build into a reusable system you can invoke whenever you decide to master a new topic.

Stay in your existing Study Partner Project. The current chat already has your full context: the Custom Instructions you generated, the four artifacts, your North Star, your learning profile. That makes it the best-positioned instance of Claude to help you design a generator for other topics.

In the same chat (or a new chat inside the same Project), paste this:

Generator prompt — build your reusable learning system
Now that my Study Partner Project is set up, I want to turn the pattern we used into a reusable Learning System Generator — something I can invoke any time I want to master a new topic in the future.

Using the context of what we just built together (the Custom Instructions, the NotebookLM artifact pipeline, the North Star framing, the knowledge map, the session structure), produce:

1. **A master prompt I can paste into a new Claude chat any time** I start a new topic. The prompt should:
   - Ask me 3–5 clarifying questions (topic, North Star, pace, prior knowledge, style)
   - Generate custom NotebookLM prompts tailored to the topic (some topics need different artifacts — a case-based topic might need a case library; a technical topic might need a glossary)
   - Produce an Encapsulation prompt for that topic's Custom Instructions
   - Give me a step-by-step install checklist

2. **A decision guide:** when to use this full system vs. when a raw Claude chat or search is fine. I don't want to over-engineer every learning task.

3. **A maintenance protocol:** how do I check in on any Study Partner after 2 weeks and adjust its Custom Instructions if pacing is off?

Output the master prompt as a clean, self-contained block I can save. If helpful, also offer to output everything as a single markdown file (learning-system-generator.md) I can download and save for later reuse.

Ask me any clarifying questions first before drafting.

Save the output somewhere you can find it — a notes app, a password manager, or a markdown file on your computer. Next topic you want to master, you're about 20 minutes from a working tutor.
